# Bug Reporting – Manual Testing Project

## 📌 Project Overview
This project focuses on identifying and reporting UI and functional bugs in a sample web application() using standard QA practices.  
The goal of this project is to understand real-world bug reporting, defect documentation, and severity/priority classification.

## 🧪 Application Under Test
- **Website:** DemoBlaze (https://www.demoblaze.com/)
- **Type:** Demo E-commerce Web Application

## 🎯 Objectives
- Identify UI and functional issues
- Document bugs using standard bug report format
- Attach screenshots as evidence
- Understand bug lifecycle, severity, and priority

## 🛠 Tools Used
- **Browser:** Google Chrome  
- **Operating System:** Windows 11  
- **Screenshot Tool:** Nimbus Screenshot  
- **Documentation:** Excel  
- **Version Control:** Git & GitHub  

## 🐞 Bugs Covered
The following types of bugs were identified:
- Product description text not properly truncated on product listing page
- Contact form allows submission with empty mandatory fields
- Cart page does not display product quantity
- Footer email is not clickable and contains dummy data
- Background overlay not removed after closing modal popup

## 📋 Bug Report Fields
Each bug report contains:
- Bug ID  
- Title  
- Module  
- Severity  
- Priority  
- Environment  
- Steps to Reproduce  
- Expected Result  
- Actual Result  
- Screenshot Reference
